# AZ-500: Microsoft Azure Security Technologies (Synthetic Guide)

## Overview

This synthetic guide outlines focus areas for the AZ-500 certification, aimed at
Security Engineers responsible for implementing security controls across Azure
environments. This document is for demonstration purposes only and does not
represent real certification content.

## Skill Area: Identity Management

Candidates should understand how to configure identity protection policies,
manage conditional access rules, and design access reviews for privileged
accounts.

## Skill Area: Platform Protection

This area covers configuring network security groups, designing perimeter
security with firewalls and gateways, and implementing host-based protections
for virtual machines and containers.

## Skill Area: Security Operations

Candidates should be able to configure centralized logging, design alert rules
for suspicious activity, and use security posture dashboards to prioritise
remediation work.

## Skill Area: Data and Application Security

This area covers encrypting data at rest and in transit, managing secrets in a
key vault service, and securing application configuration and credentials.

## Recommended Study Pattern

- 1-2 hours of focused study per day
- Weekly practice assessment checkpoints
- Target a practice score of 75% or higher before scheduling the exam
- Recommended total study time: approximately 24 hours
